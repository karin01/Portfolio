# 포트폴리오 정적 웹 구축 기록

## 목적

만들어 둔 프로그램·사이트 링크를 한 페이지에서 소개하는 **정적 포트폴리오**입니다.

## 위치

- 폴더: `Portfolio/`
- 진입: `Portfolio/index.html` 을 브라우저로 열기

## 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 뼈대, 히어로·프로젝트·소개·푸터 |
| `css/style.css` | 다크 테마, 카드·필터 스타일 |
| `js/main.js` | `portfolioProjects` 배열로 카드 생성·필터 |
| `network-dashboard.html` | 네트워크 장치 데모 대시보드 (IP 목록·차트) |
| `js/network-dashboard.js` | 대시보드 로직 · **@Network-IP-Search** (IP 열 부분 일치 필터) |
| `css/network-dashboard.css` | 대시보드 다크 UI · 검색창 스타일 |
| `assets/*.png` | 프로젝트 스크린샷(카드 썸네일·갤러리·라이트박스) |

## 수정 방법

1. `js/main.js` 의 `portfolioProjects` 에 객체를 추가하거나 `url` / `title` / `description` 을 수정합니다.
2. `type` 은 `"live"`(운영) 또는 `"local"`(로컬) — 상단 필터와 태그에 반영됩니다.
3. 브라우저로 열 URL 이 없으면 `url: null` 과 `urlCaption`(한 줄 안내)을 쓰면 됩니다.
4. 스크린샷: `thumb: "assets/파일.png"` 를 넣으면 카드 상단에 표시됩니다. `gallery: ["assets/…"]` 로 추가 썸네일을 넣을 수 있으며, **스크린샷 크게 보기** 버튼과 갤러리 썸네일이 라이트박스를 엽니다.
5. 여러 링크 버튼: `links: [{ url, label }, ...]` 를 넣으면 카드 하단에 여러 링크를 동시에 노출할 수 있습니다. (예: 프론트/엔진/대시보드)
6. GitHub: 메인 `url` 은 저장소 페이지, `repoUrl` 에 ZIP 등 다른 주소를 두면 하단에 **main 브랜치 ZIP** 같은 보조 링크가 생깁니다. `repoLinkLabel` 로 문구를 바꿀 수 있습니다.

**참고:** 스크린샷 원본은 Cursor 쪽 `assets` 에서 `Portfolio/assets/` 로 복사해 두었습니다. 새 이미지를 넣을 때도 이 폴더에 두고 경로만 맞추면 됩니다.

## 배포 아이디어

- **GitHub Pages**: `Portfolio` 를 저장소 루트 또는 `/docs` 로 올리면 무료 호스팅 가능합니다.
- **Java** 를 쓰고 싶다면: 이 페이지는 정적 파일만으로 충분합니다. Spring Boot 에서 `src/main/resources/static` 에 같은 파일을 넣어 서빙하는 방식으로 옮길 수 있습니다.

## 참고

- `localhost` 링크는 본인 PC에서만 열립니다. 외부 방문자에게 보이게 하려면 해당 프로젝트를 실제 도메인에 배포한 뒤 URL 을 바꾸면 됩니다.
